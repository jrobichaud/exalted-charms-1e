## Context

The corpus under `docs/markdown/` was generated by `scripts/convert_mermaid.py` (per the archived `mmd-markdown-conversion` change). Each file has the structure:

```
```mermaid
flowchart TD
    ... nodes with internal-link hrefs ...
```

## Charm Name
Cost: ...
Duration: ...
Type: ...
Minimum X: N
Minimum Essence: N
Prerequisite Charms: <comma-separated charm names>

<description paragraphs>


## Next Charm
...
```

There are 149 files containing 1,567 `Prerequisite Charms:` occurrences (977 unique name strings). The mermaid block already encodes prereq edges as graph arrows, so it is left untouched — this change touches only the prose `Prerequisite Charms:` lines below each `## ` heading.

Exploration found seven structural patterns the script must handle:

1. **Exact same-file** match (most common): `Wise Arrow` → `## Wise Arrow` in the same file.
2. **Exact cross-file** match: `Flesh-Mending Discipline` → `## Flesh-Mending Discipline` in `Abyssal Daybreak Medicine.md`.
3. **Punctuation/case/whitespace drift**: `Wolf Endurance Method` vs heading `Wolf-Endurance Method`.
4. **Singular/plural drift**: `Fists of Iron Technique` vs heading `Fist of Iron Technique`.
5. **Paren-suffix drift**: `Astrology Interruption Method` vs heading `Astrology Interruption Method (HUA JI)` (SMA Chinese subtitles).
6. **`(Placeholder)`-style headings**: `Heightened Smell and Taste Method` vs heading `Heightened (Sense) Methods` (slot can be multi-word).
7. **Meta-references**: `All three (Material) Withering Method Charms` — needs prefix/suffix stripping to find the core charm.

Plus three line-level format issues:
- Multi-line continuation: `Prerequisite Charms: A,\nB` should be treated as one logical reference list.
- Embedded `<br>` artifacts mid-name (e.g. `Shaping the Once-Living<br> Form`) from converter line wrapping.
- Comma-containing canonical names (currently only `One Weapon, Two Blows`) that must not be split.

Duplicate-name charms exist (5 `Ox-Body Technique` headings, 5 `Terrestrial Circle Sorcery`, 4 `Celestial Circle Sorcery`, 2 `Entombed Mind Technique`). User policy: treat each occurrence as a distinct charm — never link across files for a duplicate-name.

After ~5 iterations of source-side typo fixes by the user, the remaining unresolvable set is ~12 references: 8 generic count meta-refs ("Any three Lore Charms" — no real target exists), and ~4 charm names that are truly absent from the corpus (probably from sourcebooks not yet imported).

## Goals / Non-Goals

**Goals:**
- Resolve ≥99% of the 1,567 `Prerequisite Charms:` occurrences to valid Obsidian wikilinks.
- Leave the mermaid fenced block at the top of each file untouched.
- Emit a reviewable report of every reference the cascade could not resolve, with file and line.
- Be deterministic: rerunning on the same input produces the same output (idempotent — running twice does not double-wrap links).
- Operate in-place under `docs/markdown/`; the user relies on git for rollback.

**Non-Goals:**
- Fix source data typos. The user has already done most source-side cleanup; the script never modifies prereq text beyond wrapping in `[[]]`.
- Touch `.mmd` source files or the Sphinx build.
- Modify any other line in the markdown (description paragraphs, stat lines like `Cost:` / `Duration:` / `Minimum:` / etc.).
- Rewrite the existing mermaid `<a class='internal-link' href='#X'>` anchors. Those serve the embedded diagram and have their own (separate) cross-file semantics.
- Add new heading definitions for missing charms.
- Be a long-running daemon, watcher, or CI integration. This is a one-shot script.

## Decisions

### Decision 1: Build a heading index once, use it for all files

A single pass over `docs/markdown/**/*.md` builds:

```
HEADINGS_BY_FILE:  {file_path → set(canonical_name)}
HEADING_LOCATIONS: {canonical_name → list(file_path)}     # for cross-file lookup
NORM_INDEX:        {normalized_form → canonical_name}     # for fuzzy match
NORM_PLURAL_INDEX: {normalized_no_trailing_s → canonical_name}
PAREN_STRIPPED:    {base_form → canonical_name_with_(suffix)}
PLACEHOLDER_PATTERNS: list[(compiled_regex, canonical_name)]
COMMA_HEADINGS:    set(canonical_name containing a literal comma)
DUPLICATE_NAMES:   set(canonical_name appearing in >1 file)
```

Normalization: lowercase, drop everything that isn't `[a-z0-9]`. The plural form additionally strips trailing `s` from each word boundary before normalizing. The placeholder pattern compiles `Heightened (Sense) Methods` to `^heightened [a-z ]+? methods?$` (case-insensitive, non-greedy multi-word slot, optional final `s`).

**Alternatives considered:**
- Per-file index built lazily — rejected: every file needs the global index for cross-file links, so we'd build it anyway.
- Use a fuzzy-distance library (Levenshtein, ratio) — rejected: introduces a dependency and is non-deterministic at boundary cases; the curated cascade catches every drift class observed in the corpus.

### Decision 2: Cascade resolution, ordered from strict to permissive

For each prereq name `N`, applied in this order, first match wins:

1. `N in HEADINGS_BY_FILE[current_file]` → `[[#N]]`
2. `N in HEADING_LOCATIONS` AND `N not in DUPLICATE_NAMES` AND exactly one file → `[[OtherFile#N|N]]`
3. `normalize(N) in NORM_INDEX` → resolve to the canonical, then re-apply 1/2 to choose link form
4. `normalize_plural(N) in NORM_PLURAL_INDEX` → same
5. `N in PAREN_STRIPPED` (i.e. heading has `(SUBTITLE)` suffix and base matches `N`) → link to full canonical, alias = `N`
6. Some `PLACEHOLDER_PATTERNS[i]` matches `N` → link to that pattern's canonical heading, alias = `N`
7. Meta-strip `N` (apply `^All (three|five|two|four|\d+) `, ` Charms?$`, `^Appropriate `, ` for the appropriate \w+$` substitutions) → if the core resolves through 1–6, link with alias = original `N`
8. Otherwise → leave `N` plain text, record in report

Rules 3, 4, 5 produce a "silent canonicalize" link with no alias when the normalized form maps to a canonical that differs only by punct/case/plural/paren — per Fork 1 (Option A). Rules 6 and 7 always use an alias to preserve the original wording — per Fork 2.

**Same-file precedence for duplicates**: rule 2 explicitly checks `N not in DUPLICATE_NAMES` before allowing a cross-file link. For duplicate names, only rule 1 (same-file exact) or the meta-strip path (if it lands on a unique heading) can succeed. Otherwise the reference is left plain.

**Alternatives considered:**
- Single-pass regex replace — rejected: cannot encode the cascade's ordered fallbacks.
- ML/similarity scoring — rejected per Decision 1.

### Decision 3: Line-level preprocessing before splitting

For each file, scan top-down. When encountering `^Prerequisite Charms:` (with or without a space after the colon):

1. Capture the body after the colon.
2. **Continuation**: while the body's trailing non-whitespace is a `,` and the next file line is not blank and is not a known field header (`Cost:`, `Duration:`, `Type:`, `Minimum`, `Prerequisite Charms:`, `## `, fenced-block delimiter), append that next line (separated by a space) and consume it.
3. **`<br>` strip**: replace `<br>` (with optional surrounding whitespace) with a single space; collapse runs of whitespace.
4. **Comma-aware split**:
   - For each `name in COMMA_HEADINGS`: if `name` appears as a substring, temporarily replace it with a placeholder token.
   - Split on `,`. Strip each piece.
   - Restore the placeholder tokens back to the original name in their pieces.
5. Skip pieces equal to `None` or empty; skip pieces already starting with `[[` (idempotency).

This handles the four pathological cases observed: trailing-comma wrap, `<br>` artifact, embedded comma in `One Weapon, Two Blows`, and existing pre-converted cross-file links.

### Decision 4: Output rewriting and idempotency

The rewritten prereq line replaces the original line(s) with a single logical line of the form:

```
Prerequisite Charms: <linked piece 1>, <linked piece 2>, ...
```

A piece is rendered as one of:
- `[[#Canon]]` — same-file exact (rule 1 or 3/4 canonicalized to same-file)
- `[[OtherFile#Canon|Display]]` — cross-file (rule 2 or canonicalized form). Display = original prereq text.
- `[[#Canon|Display]]` — same-file with alias (rules 6, 7 — meta-ref or placeholder where original text differs from canonical).
- `Display` — plain text (unresolved, rule 8). Recorded in report.

For idempotency, a piece that is already a wikilink (`[[...]]`) is passed through unmodified. The continuation rule above ensures lines that were originally wrapped get joined into one line of output (no need to preserve the source's accidental wrap).

A space is always inserted after `Prerequisite Charms:` even if the source omitted it (a handful of files use `Prerequisite Charms:Name` with no space).

### Decision 5: Report file format

`docs/markdown/.prereq_report.md` is a generated index of unresolved references, structured for quick review:

```markdown
# Prerequisite Link Conversion — Unresolved References

Generated by scripts/link_prerequisites.py.

## Generic count meta-refs (no link target exists)

- `Any three Lore Charms` — dragon_blooded/Dragon-Blooded Air Lore.md:416
- ...

## Genuinely missing from corpus

- `Geas` — spirits/Spirits Demon.md:21
- `Memory Transference` — spirits/Spirits Demon.md:21
- ...
```

The script categorizes each unresolved name with a simple heuristic: lines matching `^(Any|one|One complete)\b` → generic-meta bucket; everything else → missing-or-typo bucket. The user can review and add fixes in source.

The leading `.` in `.prereq_report.md` keeps it out of Obsidian's main view in folders that don't show dotfiles.

### Decision 6: In-place rewrite, no flags

The script takes no command-line arguments. It computes `docs/markdown/` relative to itself (`script_dir / ".." / "docs" / "markdown"`) and rewrites every `.md` file there in-place. Rollback is `git checkout -- docs/markdown/`. Re-running on already-converted output is safe per Decision 4's idempotency.

**Alternatives considered:**
- `--dry-run` flag — rejected: trivial to add later if needed; for a one-shot conversion the user can do `git diff` after the run.
- Write to a new directory — rejected: would diverge from the existing in-place convention used by `convert_mermaid.py`.

## Risks / Trade-offs

- **[Risk] Over-eager fuzzy match wraps the wrong charm.** → Mitigation: the cascade is ordered strict-first, exact matches win. Normalization (rule 3) is constrained to punct/case/whitespace differences only. The 977 unique names already iterated through this cascade in exploration and produced zero false-positive matches; the plural-aware rule (rule 4) matched only 7 cases and each was verified human-correct in conversation.

- **[Risk] Placeholder regex matches too greedily.** → Mitigation: the slot pattern is `[A-Za-z ]+?` (non-greedy), and the heading text outside the slot is `re.escape`-anchored at both ends. The remaining concern is a single-word match against a multi-word slot — but since the slot is non-greedy and the regex is fully anchored, no observed reference produced an ambiguous match.

- **[Risk] Comma-containing heading list grows in the future.** → Mitigation: `COMMA_HEADINGS` is derived dynamically from the heading index every run, so any future `## Foo, Bar` heading is picked up automatically.

- **[Trade-off] No `--dry-run`.** → Acceptable: user has git, the diff is the dry-run output.

- **[Trade-off] In-place modification touches every file with a prereq line.** → Acceptable: the change is reviewable as a single commit; the diff for each file is bounded to its `Prerequisite Charms:` lines.

- **[Risk] Mermaid block accidentally rewritten.** → Mitigation: the script's file loop tracks whether the cursor is inside a fenced ` ```mermaid ` block (toggling on opening and closing fences) and skips rewriting while inside.

## Migration Plan

1. Implement `scripts/link_prerequisites.py` per the cascade above.
2. Run once: `python scripts/link_prerequisites.py`.
3. Review `git diff` and `docs/markdown/.prereq_report.md`.
4. If the report surfaces a fixable source typo (the user has been doing this iteratively), fix the source and rerun until the report is stable.
5. Commit the linked markdown + report file.

Rollback at any point: `git checkout -- docs/markdown/` and delete `docs/markdown/.prereq_report.md`.
